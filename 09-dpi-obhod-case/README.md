# 🛡️ Кейс: Обход DPI/ТСПУ на прикладном уровне (L7) в Ubuntu 26.04

> **Системный аналитический подход к решению сетевой задачи**

---

## 📌 Описание проблемы

В условиях ограниченного доступа к ресурсам (YouTube, GitHub, DeepSeek, Leonardo AI) требуется решение, которое:

- ✅ Не влияет на работу системы (apt, обновления)
- ✅ Позволяет выборочно направлять трафик через обход
- ✅ Легко развёртывается и автоматизируется
- ✅ Не требует постоянного вмешательства пользователя

---

## 🧠 Архитектура решения

### Ключевая идея

Обход DPI поднимается как **локальный SOCKS5-прокси** на порту `1080`.  
Вся система продолжает работать напрямую, а обход используется только для заблокированных сайтов (через браузер или приложения, поддерживающие прокси).

```mermaid
graph LR
    A[Браузер] -->|SOCKS5| B[127.0.0.1:1080]
    B --> C[tpws --mode 1 --midsld --disorder --oob]
    C --> D[Фрагментация TLS]
    D --> E[Интернет]
    E --> F[YouTube / GitHub / DeepSeek]
```

---

🛠️ Технология

Используется tpws с комбо-стратегией:

```bash
tpws --mode 1 --midsld --disorder --oob
```

Параметр Эффект
--mode 1 Перехват и модификация TLS-пакетов
--midsld «Ослепление» DPI путём маскировки метаданных
--disorder Перемешивание TCP-пакетов для сбивания анализаторов
--oob Внедрение Out-of-Band данных

🔧 Установка и запуск

One-Line Deploy

```bash
curl -kLO https://raw.githubusercontent.com/k0sm0/dpi-obhod/main/install_dpi.sh && chmod +x install_dpi.sh && ./install_dpi.sh
```

---

🧪 Настройка браузеров

Firefox

1. Настройки → Параметры сети → Ручная настройка прокси
2. SOCKS v5: 127.0.0.1, Порт: 1080
3. ✅ Включить «Проксировать DNS»

Brave / Chrome

```bash
brave --proxy-server="socks5://127.0.0.1:1080"
```

Внутри браузера:

· Открыть brave://flags
· Найти Experimental QUIC protocol → Disabled
