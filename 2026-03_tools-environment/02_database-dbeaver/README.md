# 🗄️ Практика: Работа с базами данных и анализ данных в СУБД (DBeaver & SQL)

## 📌 Паспорт модуля
* **Цель**: Научиться использовать универсальный менеджер баз данных DBeaver для подключения к реляционным СУБД (PostgreSQL), анализа физической структуры таблиц, проверки качества данных и написания SQL-скриптов.
* **Стек технологий**: PostgreSQL, DBeaver, SQL (DQL), операторы фильтрации и агрегации (`SELECT`, `WHERE`, `GROUP BY`, `HAVING`, `COUNT`).

## 🔌 1. Подключение и обходные пути в DBeaver

Для проведения системного анализа и проверки данных аналитик настраивает прямое подключение к тестовому контуру базы данных через параметры хоста (`Host`), порта (`Port`), имени БД (`Database Name`) и учетных данных авторизации, предоставленных администратором.

### Лайфхак для ускорения рутины (Генерация SQL):
Для быстрого анализа структуры данных и экономии времени на ручном написании шаблонного кода используется встроенная функциональность DBeaver:
1. Аналитик кликает правой кнопкой мыши по целевой таблице в навигаторе (например, `users`).
2. Выбирает пункт меню **Generate SQL** \(\rightarrow\) **SELECT**.
3. Программа автоматически формирует готовый синтаксический каркас запроса со всеми физическими столбцами таблицы.
4. Скрипт копируется в SQL-редактор, где аналитик кастомизирует его под бизнес-задачи.

## 📜 2. Проверка качества данных и SQL-скрипты

В рамках задач по миграции данных, интеграции или поиску системных багов аналитик использует SQL для выявления аномалий и нарушений бизнес-логики.

### Кейс №1: Поиск дубликатов в критических полях
При проектировании систем регистрации часто возникает баг, когда из-за отсутствия уникального индекса (`UNIQUE constraint`) пользователи регистрируют разные аккаунты на одну и ту же почту. Для выявления таких дубликатов в таблице `users` используется следующий запрос:

```sql
SELECT email, COUNT(*) AS duplicate_count
FROM users 
WHERE email IS NOT NULL
GROUP BY email 
HAVING COUNT(*) > 1;
```

#### Разбор логики запроса для команды разработки:
* `GROUP BY email` — схлопывает (группирует) все строки таблицы по значениям в столбце электронной почты.
* `COUNT(*)` — подсчитывает точное количество записей в каждой образовавшейся группе.
* `HAVING COUNT(*) > 1` — жестко фильтрует сгруппированные данные, выводя на экран только те email, которые встретились в базе данных более одного раза.
